# Environment Variables Setup

## Configuration

The Rainforest API is now configured to use environment variables for secure credential management.

### Files Created

1. **`.env`** - Contains your actual API credentials (⚠️ **Never commit this file**)
2. **`.env.example`** - Template file showing the required environment variables structure

### Environment Variables

The following environment variables are used:

- `EXPO_PUBLIC_RAINFOREST_API_KEY` - Your Rainforest API key
- `EXPO_PUBLIC_RAINFOREST_API_URL` - The Rainforest API endpoint URL

> **Note:** Expo requires the `EXPO_PUBLIC_` prefix for environment variables to be accessible in your app code.

### Setup Instructions

1. Copy `.env.example` to `.env` (already done)
2. Update the `EXPO_PUBLIC_RAINFOREST_API_KEY` value in `.env` with your actual API key
3. **Restart your Expo development server** for changes to take effect:
   ```bash
   # Stop the current server (Ctrl+C)
   # Then restart with:
   npm start
   ```

### Security

- The `.env` file is already added to `.gitignore` to prevent committing sensitive credentials
- Always use `.env.example` as a template for other developers without exposing your actual keys
- Never hardcode API keys directly in your source code

### Usage

The API configuration is automatically loaded from environment variables in `services/rainforestApi.ts`. The app will throw an error if the API key is not properly configured.
